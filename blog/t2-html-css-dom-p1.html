HTML, CSS & DOM Part 1

  • Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

  • HTML is like the building materials. CSS is like the blue prints. and DOM is like the standards you follow.

  • What is meant by boxifying design?

  • When designing a webpage a developer should keep in mind that a webpage is presented as a set of boxes in the browser. When designing a webpage, a developer should cut up thier design into boxes to better understand where and how elements will be displayed. Within those boxes are the elements which are displayed. No matter the shape that can be seen on the webpage, it is still within a box in the end. Boxes may have transparent parts as to display circles and other non-box shapes.

  • What is the box model?

  • The box model is a way to visualize how object will show up in a browser. It contains the margins, paddings, borders and the object itself. It helps when you're trying to position things correctly on the webpage and simply make things look nice.
